5. DESPLIEGUE LOCAL DEL BACKEND

GUÍA PARA EJECUTAR EL BACKEND LOCALMENTE EN CASO DE FALLOS EN RAILWAY.

REQUISITOS PREVIOS

  • NODE.JS (VERSIÓN 18 O SUPERIOR): DESCÁRGALO DE nodejs.org
  • POSTGRESQL (BASE DE DATOS LOCAL): INSTALA DESDE postgresql.org O USA DOCKER
  • GIT: PARA CLONAR EL REPOSITORIO

INSTALACIÓN DE POSTGRESQL CON DOCKER (OPCIONAL)

SI PREFIERES DOCKER:

docker run --name postgres-pokequiz -e POSTGRES_PASSWORD=mypassword -e POSTGRES_DB=pokequiz_db -p 5432:5432 -d postgres:15

LUEGO USA DATABASE_URL="postgresql://postgres:mypassword@localhost:5432/pokequiz_db"

PASOS DE INSTALACIÓN

  1. CLONAR EL REPOSITORIO DEL BACKEND:
git clone https://github.com/GaizkaDM/PokequizBackend.git
cd PokequizBackend
  1. INSTALAR DEPENDENCIAS:
npm install
  1. CONFIGURAR LA BASE DE DATOS:

COPIA EL ARCHIVO .env Y EDITA LAS VARIABLES NECESARIAS:

cp .env .env.local  # O edita directamente .env
# Edita las siguientes líneas:
DATABASE_URL="postgresql://usuario:password@localhost:5432/pokequiz_db"
JWT_SECRET="tu_clave_secreta_segura_aqui"
PORT=3001
  1. EJECUTAR LAS MIGRACIONES DE PRISMA:
npx prisma migrate dev
  1. GENERAR EL CLIENTE DE PRISMA:
npx prisma generate
  1. POBLAR LA BASE DE DATOS CON DATOS INICIALES:
npm run seed
  1. INICIAR EL SERVIDOR:
npm start

EL BACKEND ESTARÁ DISPONIBLE EN http://localhost:3001

CONEXIÓN DEL FRONTEND

MODIFICAR EL ARCHIVO Network.kt EN EL PROYECTO ANDROID PARA APUNTAR AL BACKEND LOCAL:

const val BASE_URL = "http://10.0.2.2:3001/"
USA 10.0.2.2 EN LUGAR DE localhost PARA EL EMULADOR DE ANDROID.

SOLUCIÓN DE PROBLEMAS

  • SI EL PUERTO 3001 ESTÁ OCUPADO, CAMBIA EL PORT EN .env
  • VERIFICA QUE POSTGRESQL ESTÉ EJECUTÁNDOSE
  • REINICIA EL EMULADOR DESPUÉS DE CAMBIAR LA URL
