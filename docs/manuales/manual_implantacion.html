Manual Técnico de Implantación

Este documento constituye el manual técnico oficial para la implantación, configuración y despliegue de la aplicación Pokequiz.

1. Ficha Técnica del Proyecto

  • Plataforma: Android (Nativo)
  • Lenguaje: Kotlin
  • Framework UI: Jetpack Compose
  • Arquitectura: Basada en componentes con gestión de estado reactiva
  • SDK Mínimo: 29 (Android 10.0)
  • SDK Objetivo: 36 (Android 15+)

2. Arquitectura del Sistema

2.1. Frontend (Android)

Construido con Jetpack Compose y estética "Retro GBA". Utiliza Retrofit 2 para networking y Coil para procesamiento de imágenes.

2.2. Backend

Servidor en Node.js con Express y Prisma ORM, conectado a una base de datos PostgreSQL alojada en Railway.

3. Requisitos del Entorno

  • IDE: Android Studio (Ladybug o superior).
  • JDK: Java Development Kit 11.
  • Hardware: Dispositivo físico o Emulador API 29+.

4. Configuración

4.1. Clonación

git clone <url-del-repositorio>

4.2. local.properties

sdk.dir=C\:\\Users\\<Usuario>\\AppData\\Local\\Android\\Sdk

5. Integración con Servicios

IMPORTANTE: La URL base del servidor está en Network.kt. 
const val BASE_URL = "https://pokequizbackend-production.up.railway.app/"

6. Despliegue

Para generar el binario final, use Build > Generate Signed Bundle / APK... en Android Studio seleccionando la variante release.

7. Troubleshooting

Problema Solución
Error de Red Verificar estado del backend en Railway.
Fallo SDK Instalar Android API 36 en SDK Manager.
Gradle Sync Error Actualizar plugin de Kotlin en build.gradle.
